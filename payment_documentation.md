# Payment & Subscription Documentation

This document outlines the process for a user (Business) to view available membership tiers and subscribe to a plan using the Payment resource.

## 1. View Available Tiers

Before making a payment, the user needs to select a tier. You can retrieve all available tiers using the following public endpoint.

### Endpoint
`GET /tier`

### Access
Public

### Response
Returns an array of `Tier` objects.

```json
[
  {
    "id": "d0b3b3e0-0b3e-4b3e-8e3e-3e0b3b3e0b3e",

  ...
]
```

## 2. Subscribe to a Tier

Once a tier is selected, the user can subscribe to it using the `subscribe` endpoint. This endpoint supports both Stripe and PayPal providers.

### Endpoint
`POST /payment/subscribe`

### Access
**Role:** Business (User must be logged in as a Business)

### Request Body (`SubscribeDto`)

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `tier_id` | `string` | Yes | The UUID of the tier to subscribe to. |
| `plan_type` | `enum` | Yes | The billing cycle. Values: `MONTHLY`, `QUARTERLY`, `ANNUALLY`. |
| `provider` | `enum` | No | The payment provider. Values: `stripe`, `paypal`. Default: `stripe`. |
| `payment_token` | `string` | **Yes** (Stripe) | The payment token generated by the frontend (e.g., Stripe Elements). Required if `provider` is `stripe`. |
| `return_url` | `string` | **Yes** (PayPal) | The URL to redirect the user to after a successful PayPal payment. Required if `provider` is `paypal`. |
| `cancel_url` | `string` | **Yes** (PayPal) | The URL to redirect the user to if they cancel the PayPal payment. Required if `provider` is `paypal`. |
| `is_trial` | `boolean` | No | Whether this is a trial subscription. Default: `false`. |

#### TypeScript Interface

```typescript
export enum PlanType {
  MONTHLY = 'MONTHLY',
  QUARTERLY = 'QUARTERLY',
  ANNUALLY = 'ANNUALLY',
}

export enum PaymentProvider {
  STRIPE = 'stripe',
  PAYPAL = 'paypal',
}

export interface SubscribeDto {
  tier_id: string;
  plan_type: PlanType;
  provider?: PaymentProvider;
  payment_token?: string; // Required for Stripe
  return_url?: string;    // Required for PayPal
  cancel_url?: string;    // Required for PayPal
  is_trial?: boolean;
}
```

### Example Requests

#### Stripe Subscription
```json
{
  "tier_id": "d0b3b3e0-0b3e-4b3e-8e3e-3e0b3b3e0b3e",
  "plan_type": "MONTHLY",
  "provider": "stripe",
  "payment_token": "tok_visa"
}
```

#### PayPal Subscription
```json
{
  "tier_id": "d0b3b3e0-0b3e-4b3e-8e3e-3e0b3b3e0b3e",
  "plan_type": "ANNUALLY",
  "provider": "paypal",
  "return_url": "https://myapp.com/payment/success",
  "cancel_url": "https://myapp.com/payment/cancel"
}
```

### Response

#### Success (201 Created)

The response structure depends on the provider but generally indicates a successful subscription creation.

**Stripe Example:**
```json
{
  "status": "active",
  "clientSecret": "pi_...",
  "subscriptionId": "sub_..."
}
```

**PayPal Example:**
```json
{
  "status": "APPROVAL_PENDING",
  "orderId": "...",
  "approveLink": "https://www.sandbox.paypal.com/checkoutnow?token=..."
}
```
*Note: For PayPal, the frontend must redirect the user to the `approveLink`.*
